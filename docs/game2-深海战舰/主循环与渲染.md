## 2. 线程循环（`a.run()`，a.java:774-791）

```java
long t0 = currentTimeMillis();
while (this.w != null) {
    if (this.s && !this.t) {            // s=运行标志, t=正在paint(防重入)
        long dt = currentTimeMillis() - t0;
        if (dt < 80) Thread.sleep(80 - dt);   // 定帧 80ms
        repaint();                      // 触发 paint()
        t0 = currentTimeMillis();
    }
}
```

- **帧节奏：目标 80ms/帧 ≈ 12.5 FPS**（游戏1 是 100ms）。不足 80ms 则 sleep 补齐；超时不丢帧也不补偿。
- `s`（a.java:54）=线程运行；`t`（a.java:55）=`paint` 进行中，置位期间循环跳过 `repaint`，避免重入。
- `repaint()` 异步请求系统回调 `paint(Graphics)`；逻辑更新**全部在 paint 内完成**（无独立 update 线程）。
- 线程在构造时启动（a.java:81-82），`destroyApp` 时 `w=null` 退出（GameMIDlet.java:76）。

> ⚠️ 复刻建议：TS+Canvas 用 `requestAnimationFrame` + 80ms 累加器固定步长（fixed timestep），每步执行一次 paint 等价逻辑。逻辑与渲染耦合在一帧内，无插值。

## 3. paint 总流程（`a.paint(Graphics)`，a.java:126-550）

每帧开头（无论何状态）：
1. `this.i++`（a.java:127）——全局帧计数（用于 `i%4` 闪烁节拍）。
2. `this.t=true`（a.java:128）——置 paint 锁。
3. `setFont(Font.getFont(0,0,8))`（a.java:129）——SMALL 字体。
4. `GameMIDlet.a()`（a.java:130）——**音频帧计时**（`g--`，归零停播放器）。
5. `switch(this.b)` —— 按主状态分发（见《游戏状态机.md》）。
6. 末尾 `this.t=false`（a.java:549）——解锁。

整个 switch 包在 `try{}catch(Exception){}`（a.java:131,548）——异常被吞掉（不崩溃但可能黑屏）。

### 游戏中（`b==10`）渲染（a.java:372-375）
```java
this.z.b();        // 场景逻辑更新（物理+AI+相机+spawn）
this.z.a(graphics); // 场景渲染
```
**逻辑与渲染分两步**：先 `j.b()` 推进世界，再 `j.a(g)` 画出。

### 剧情对话过场（`b==20`）分发（a.java:327-330）
```java
case 20: { this.a(graphics); break; }   // 调私有 a(Graphics)（a.java:583）
```
`case 20` 整屏交给私有方法 `a(Graphics)`（a.java:583）绘制，不进 `j` 场景渲染（详见 §5.x）。

4. 逐个 f[i].a(graphics, camX>>10, camY>>10)  画 actor          (231-235)

1. 地图背景层（bpng，仅 Boss 关 `g==2`）
2. 地图前景瓦片（fpng）

## 5b. 瓦片层绘制（TileMap `b`）

`j.a(Graphics)` 第 1 步的 `a.a(graphics)`（地图层）即 `b` 类（TileMap）。其无参入口 `b.a(Graphics)`（b.java:73-75）只是转发：

```java
public final void a(Graphics graphics) {
    this.a(graphics, this.c, this.d, this.a, this.b);  // c,d=滚动偏移; a,b=视口宽高
}
```

- `this.c/this.d`（b.java:18-19）=当前滚动偏移，由 `j.b()` 末尾 `a.a(camX>>10, camY>>10)`→`b.a(int,int)`（b.java:55-58）写入。
- `this.a/this.b`（b.java:16-17）=视口宽高，构造时 `b(176,172)` 传入（构造器 b.java:50-53）。

### 增量重绘到离屏缓冲（`b.a(Graphics,n,n2,n3,n4)`，b.java:205-267）

地图**不每帧全画**，而是维护一张离屏 Image `F`（静态，b.java:47-48；尺寸由 `f(int,int)` 按视口+1块边距创建，b.java:146-154），按相机移动方向**只补绘新进入的边缘块列/行**：

1. 取 `F` 尺寸 `n7×n8`、缓冲 Graphics `G` 置白（b.java:208-210）。
2. 算当前相机覆盖的块对齐边界 `n9..n12`（按块尺寸 `p/q` 向下取整对齐，b.java:211-214）。
3. **首帧全绘**：`B<0`（`a()` 复位为 -1，b.java:60-63）→ 整窗调 `a(G, n9,n10,n11,n12, n7,n8)` 把整片瓦片画进 `F`，并记下边界 `B/C/D/E`（b.java:215-221）。
4. **水平差分**：`B!=n9` → 只补绘相机右进/左进的一**列**新块到 `F`，更新 `B/D`（b.java:222-233）。
5. **垂直差分**：`C!=n10` → 只补绘下进/上进的一**行**新块，更新 `C/E`（b.java:234-245）。
6. **环绕拷贝到屏幕**：`F` 是环形复用的，按相机在 `F` 内的取模坐标 `n13..n16` 把可视区分 1~4 块从 `F` blit 到目标 `graphics`（`b(graphics,...)`=`drawImage(F, n5-n, n6-n2, 20)`，b.java:200-203），覆盖跨边界的回绕情形（b.java:246-265）。
7. 末尾 `setClip(0,0,n3,n4)` 还原裁剪（b.java:266）。

### 单元绘制的层序（`b.a(G,n,n2,n3,n4,n5,n6)`，b.java:156-198）—— **关键绘制顺序**

把 `[n,n3]×[n2,n4]` 范围内每个瓦格画进缓冲 `G`，双重循环逐格（步进 `p`/`q`），每格**自底向上两层**：

1. **背景层（仅 Boss 关 `g==2`）**：`d(...)` 查 `b.bin`/`bpng.bin` 索引，`drawImage(this.y=bpng, ...)`（b.java:167-176）。`g` 在加载时由 `a(int,int,int)` 第三参写入（b.java:338-339），`g==2` 才加载并绘背景图层 `y`（b.java:339-352）。
2. **前景瓦片层**：`c(...)` 查 `p.bin` 瓦片索引（`-1`=空格跳过），`drawImage(this.r=fpng, ...)`（b.java:177-186）。

> 即**先背景层后前景瓦片层**，与文档 §5 "绘制层次"一致并细化：背景层来自 `bpng.bin`（仅 `g==2` Boss 关存在），前景瓦片来自 `fpng.bin`。两层都画进离屏 `F`，再整体 blit 到屏，**Actor/HUD 在其上层**由 `j.a(Graphics)` 绘制（j.java:205-251）。
>
> 索引计算细节：`c(n,n2)`（b.java:115-131）经 `e(n,n2)`（b.java:77-83）两级查表（图块层 `l[]` → 瓦片表 `t[]`），把世界格坐标映射到 `fpng` 图集中的子图编号；`d(n,n2)`（b.java:133-144）同理映射 `bpng`。资源装载见 `b.a(int,int,int)`（b.java:284-359）：依次读 `m.bin`(图块布局)、`p.bin`(瓦片/碰撞)、`fpng.bin`(前景图集)，`g==2` 时再读 `b.bin`+`bpng.bin`(背景)。

2. 右软键(22)：若 `b==10` 且 `z.w==0` → `e=0; b=22`（进任务简报）。
3. 其余：`this.c = d(i)`（写当前掩码）。

### keyReleased（a.java:883-887）
`c=0; d=-1`（清掩码与"持续下键"标志）。

> 掩码 `c` 是「事件型」：逻辑读取后多处主动清 0（如 `this.c=0` a.java:284、g.c(int) 内 `this.L.c=0` g.java:691 等），实现"按一次触发一次"。方向移动则靠 keyReleased 清 0 实现"按住持续"。

## 7. 相机系统（`a.a(int,int)`，a.java:698-747）

- 目标坐标由 `j.b()` 算出（跟随玩家或锁定 `j.A/B`）。
- **平滑步进**：差值 >16384(X)/10240(Y) 则按 `y` 速度追赶，否则直接对齐（a.java:701-728）。
- **震屏**：`L` 为真时按 `K%2` 左右抖 ±2048，`K` 递减到 0 停（a.java:729-734）；触发 `b(int)`（a.java:888-893）。
- **边界钳制**：相机 X∈[0, I-q]、Y∈[0, J-r]（`I/J`=地图像素总宽/高，a.java:735-746）。
- TileMap 接收 `a.a(camX>>10, camY>>10)` 做滚动增量重绘（j.java:202 → b.java:55）。
