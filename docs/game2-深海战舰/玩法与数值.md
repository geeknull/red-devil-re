### 3.1 翻越/攀爬判定 `g.t()`（g.java:857-920）

按上键时（`c(int)` 的 `case 64/128`、且处于地面态 `G&1`）调用 `t()` 探测身前障碍，返回码 1-5 决定动作与 `a[]/b[]` 初速档位（g.java:537-555）。

**探测几何（g.java:861-874）**

- 朝向决定扫描方向 `n`（右 `H==0`→+1，左→−1）与起止列（前缘 `q`/`p`，前探 10 像素）：
  - 右：列 `n4=(u>>10)+q >>3` 到 `n5=(u+w>>10)+q+10 >>3`；
  - 左：列 `n4=(u>>10)+p >>3` 到 `n5=(u+w>>10)+p−10 >>3`。
- 纵向扫描带：`n6=(v+x>>10)−2 >>3`（脚下基准格），`n7=n6−7`（向上 8 格=64 像素的检测窗）。
- 逐列（`n8`）自上而下逐格（`n9` 从 `n7` 到 `n6`）查 `a(n8,n9)`，命中实心（`&3 != 0`）即按命中高度 `n11=n9−n7`（0=最顶，6=最底）分档（g.java:879-913）：

| 命中高度 `n11` | 返回码 | 含义 / 后续动作 |
|---|---|---|
| 0（最高，齐头） | **5** | 太高翻不过，撞墙站定。`a[5]=8192,b[5]=−10240`（普通跳），切动作 `0xE(14)`，置 `aa=false`（g.java:543-547,882-884） |
| 1-2 | 空中态 → **5**；地面态 → 见下「`n11` 细分」 | 半身高：空中（`G&1==0`）撞墙 `return 5`（g.java:887-889）；地面态不 return，fall-through 进 case 3/4 攀爬（g.java:885-890） |
| 3-4 | 见下「`n11` 细分」 | 高台攀爬带 |
| 5-6（最低，齐脚） | **1** | 矮坎，直接跨过。`a[1]=4096,b[1]=−10240`（小跳）（g.java:909-911） |
| 无命中 | **0** | 无障碍。`a[0]=8192,b[0]=−10240`（默认起跳）（g.java:919） |

「`n11` 细分」（case 1/2 在地面态会 fall-through 进 case 3/4，故 `n11∈{1,2,3,4}` 共用此判定，g.java:885-908）：当身前一格 `n12=a(n8−n,n9)` 与上方一格 `n13=a(n8,n9−1)` **均空**（`==0`）时记录落点 `X/Y` 并按 `n11` 返回：`n11<3`（即 1/2）→ **4**；`n11==3` → **3**；`n11==4` → **2**。若身前或头顶被堵 → 退化返回 **4**（g.java:907）。各档对应初速（按返回码索引 `a[W]/b[W]`，g.java:549-550）：返回 **3**→`a[3]=8192,b[3]=−17408`（跳得最高），返回 **2**→`a[2]=8192,b[2]=−15360`，返回 **4**→`a[4]=8192,b[4]=−15360`。

> 说明：`switch` 各 case **无 break**（fall-through），靠 `return` 提前退出，故上表「地面态落入下一档」成立。`t()` 自上而下逐格扫描，命中的**第一格**高度即决定档位（越靠头顶=越高=翻不过=5）。

**攀爬落点 `X/Y`（g.java:896-898）**：可攀爬时算出目标坐标：
- `X = (n>0 ? n8<<13 : (n8<<3)+8<<10)`，再按朝向 ±10240（`H==0` 减、否则加）——即落到障碍格前缘并内收约 10 像素；
- `Y = (n9<<13) + 40960`——落到命中格下方 40 像素（约站立点）。

**翻越动作触发（g.java:537-555）**：
- 返回 2 或 3（高台攀爬）→ `aa=false`，`u=X`（瞬移到落点列），切动作 `0x11(17)`（攀爬态）；
- 其余（含 0/1/4/5）→ `v−=5120`（上提 5 像素），切动作 `0xE(14)`（跳/翻越态）；返回 5 时额外 `aa=false`。
- 统一赋初速 `y=a[W]×朝向, z=b[W], B=4096(重力)`，置 `ad=true`（翻越中），`G&=~1`（离地）。

**攀爬执行 `u()`（g.java:922-951）**：在 `ad && aa` 时驱动到 `X/Y`：
- 若 `d(0)` 命中可对齐门/梯（地形值 `==4`）→ 停住切动作 `0x12(18)`、`G=2`（攀爬中态）（g.java:926-931）；
- `X/Y` 未解算时补调 `t()`（g.java:933-935）；
- 已有落点：当 `|v−Y| <= |x|`（纵向接近）且头顶空 `!x()` → 直接把位移补到落点（`w=X−u, x=Y−v`，清速，置 `Z=true` 完成），否则若 `|u−X| <= |w|`（横向到位）则清水平速（g.java:936-949）。

**辅助探针**：
- `w()`（g.java:1097-1107）：检查头顶 3 格是否有实心（爬升净空）。
- `x()`（g.java:1109-1118）：检查落脚处下方 2 格是否实心（落点是否站得住）。
- `d(int)`（g.java:953-966）：检测身前是否为地形值 `==4` 的可对齐物（门/梯/通道），命中则把 `u` 吸附对齐到该列（`n==0` 用包围盒上沿 `r`，否则用固定 10）。

> 地形碰撞值语义（`a(n,n2)`=`b.b()`，h.java:347 / b.java:85-113）：返回每像素碰撞属性。`==1` 纯实心墙；`&3 != 0`（含 1、2）为可站立/可攀爬实心面；`==4` 为可对齐特殊物（门/梯/移动平台触发，见 g.java:121 的 `&4` 与 `d()`）。⚠️ 属性位 2/3 的精确美术含义（如「斜坡」「半砖」）未在源码中以常量命名，待结合 m.bin 图块表核实。

### 4.2 子弹物理主循环 `k.b()`（k.java:26）

按子弹类型 `h` 分派（`h` 为 actor type，非伤害）。共有 type 9/10/12/16，下面逐型给出 CFR 恢复后的完整逻辑。

**type10 — 直射/榴弹主体（k.java:28-141）**

- `l==9`（榴弹下坠态）：若垂直速度 `|z| <= 2048` 则切到动作 `0xA(10)`（k.java:31-33）。（`z` 为垂直分量，与 §4.1 表一致）
- `l==10`：若 `z > 3072` 则切到动作 `0xB(11)`（k.java:36-38）。
- 上述及 `l∈{0..5,11,12}` 均 `++f`（累计存活帧，k.java:48）。
- 此分支守卫为 `h==6`（actor type）`&& !b(1)`，但因身处 `type10`（`h==10`）主体内，`h==6` 恒不成立，整段 `b--`/`y=0` 实为死代码；type10 直射弹实际仅走 `else if (f>30)` 标记出界销毁 `R=true`（k.java:51-57）。

- **地形命中判定**（k.java:60-85，方向性探针）：
  - 向右飞（`y>0`）：撞墙 `a()` 真 → `d=true`；否则越过右边界 `u > L.m+L.q+8192` → `R=true`。

  - 向上飞（`z<0`）：撞顶 `l()` 真 → `d=true`；否则越过上边界 `v < L.n−8192` → `R=true`。

- **命中产生爆炸碎片**（k.java:86-90）：若（命中实体 `Q` 或命中地形 `d`）且动作 `l∈{6,9,10,11,12}` → 在 `(u,v)` 生成 type12 爆炸特效（`k.a(12,0,u,v,K,null)`）并 `e()` 自销毁。
