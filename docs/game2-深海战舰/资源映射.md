## 0. 归档容器格式（所有 .bin 通用，已实测确认）

```
[int32 count]                  小端
[int32 offset[count]]          小端，count 个偏移
[blob]                         数据区
```
- **偏移基准 = 偏移表末尾**，即 `base = 4 + count*4`。`entry[i] = blob[base+offset[i] : base+offset[i+1]]`。
- **`count` 含 1 个 EOF 哨兵**：真实条目数 = `count - 1`，最后一个 offset == 文件尾，对应"第 count-1 条"为空。
- `offset[0]` 恒为 0。
- 加载器：`GameMIDlet.b(path,i)`（定位流，GameMIDlet.java:224-244）按字段流式读；`c(path,i)`（整条读取，246-273）用于 PNG/字符串。

## 3.5 b.a() — 瓦片绘制管线：m/p/b.bin 如何映射到 PNG（据 CFR 恢复 ✅）

> 本节据 CFR 完整恢复的 `b.a(Graphics)`（b.java:73）及其依赖补全；此前 jadx 反编译失败。它把前几节的静态字段串成实际渲染流程，明确了 **m.bin → p.bin → fpng.bin** 与 **b.bin → bpng.bin** 两条引用链。

### 调用入口与缓冲合成

- `b.a(Graphics)`（b.java:73-75）：无参便捷入口，转调 `a(graphics, this.c, this.d, this.a, this.b)`，其中 `(a,b)` 是视口宽高（构造时传入，b.java:50-53）、`(c,d)` 是当前相机左上角像素坐标（`b.a(int,int)` 设置，b.java:55-58）。
- `b.a(Graphics,n,n2,n3,n4)`（b.java:205-267）：**离屏缓冲 + 增量重绘**合成器。维护一张离屏图 `F`/其 Graphics `G`（`b.f()` 按视口尺寸+1~2 个 tile 余量创建，b.java:146-154），仅当相机跨过整块 tile 边界（`B/C/D/E` 记录上次对齐坐标，b.java:215-245）才把新滚入的那一条 tile 区域用核心绘制循环画进 `F`；随后用 `b.b()`（b.java:200-203）按环绕（取模 `F` 尺寸）把 `F` blit 到屏幕，支持横纵 wrap 的 1~4 次拼贴（b.java:250-265）。

### 核心绘制循环 `a(Graphics,n,n2,n3,n4,n5,n6)`（b.java:156-198）

逐 tile 遍历可视矩形（步距 `p×q`=8×8），每个 tile 位置先画背景层、再画前景层：

1. **背景层（仅 `g==2` 的关卡 #0/#2/#5/#6（含 Boss），b.java:167-176）**
   - `n7 = this.d(n13,n11)`（b.java:169）→ 见 `d()`：直接从 `b.bin` 的 `A[]` 取背景 cell 号。
   - `graphics.drawImage(this.y, x - n7%z*w, y - n7/z*x, 20)` 配 `setClip(...,w,x)`（b.java:174-175）：把 **bpng 图集**第 `n7` 个 cell（按步距 `z` 拆 (列,行)）裁到当前格。
2. **前景/地形层（恒执行，b.java:177-186）**
   - `n7 = this.c(n13%n9, n11%n10)`（b.java:178，`n9=b()`地图像素宽、`n10=c()`地图像素高，循环平铺取模）→ 见 `c()`：经 m.bin→p.bin 解析出 fpng cell 号。
   - `n7 != -1` 时 `graphics.drawImage(this.r, x - n7%s*p, y - n7/s*q, 20)` 配 `setClip(...,p,q)`（b.java:184-185）：把 **fpng 图集**第 `n7` 个 cell（按步距 `s` 拆 (列,行)）裁到当前格；`-1` 表示空块/空 tile，跳过。

### 索引解析：从地图坐标到 PNG cell

- **`e(x,y)`（b.java:77-83）地图→块**：`l[ y/(i·q)·j + x/(h·p) ]`，即按块尺寸（`h·p × i·q` = 4·8×4·8 = 32×32 像素）取 `m.bin` 块网格 `l[]` 的块号（255→-1 视为空白格，b.java:79-81）。
- **`c(x,y)`（b.java:115-131）→ fpng cell（前景链）**：先 `e()` 得块号 `n4`；块在调色图中按 `n/(n.h)` 列数排布，定位到块内 tile：`col = n4%(n/h)·h + x/p%h`、`row = n4/(n/h)·i + y/q%i`（b.java:119-120），再查 `p.bin` 的 tile→fpng 映射表 `t[ row·n + col ]`（b.java:121-122）得到 **fpng 图集上的线性 cell 号**（负+256，b.java:123-125）。这条链即 **m.bin 块网格 → p.bin `t[]` → fpng.bin 像素**。
- **`d(x,y)`（b.java:133-144）→ bpng cell（背景链）**：背景独立平铺，直接 `A[ y%(x·v)/x·u + x%(w·u)/w ]`（b.java:135-137，`u/v`=背景列/行 tile 数，`w/x`=背景 tile 像素 8×8）得到 **bpng 图集上的线性 cell 号**。这条链即 **b.bin `A[]` → bpng.bin 像素**（不经 m/p.bin）。

### 引用链小结

| 层 | 几何来源 | 内容索引来源 | 间接表 | 最终 PNG | cell 步距 |
|---|---|---|---|---|---|
| 前景/地形 | `m.bin` 块网格 `l[]`（`e()`） | `p.bin` 映射表 `t[]`（`c()`） | 块→tile→`t[]` | `fpng.bin`[`p.bin.fpngIdx`] = `r` | `s = r.width / p`（b.java:337） |
| 背景（仅 `g==2`） | `b.bin` 块表 `A[]`（`d()`） | 同 `A[]`（直接即 cell 号） | 无 | `bpng.bin`[0]（恒第 0 条）= `y` | `z = y.width / w`（b.java:351） |

> 与第 10 节加载触发点一致：`b.a(mapId,bgId,bA)`（b.java:284）依次读 m.bin[mapId] → p.bin[palIdx] → fpng.bin[fpngIdx]，并在 `bA==2` 时追加 b.bin[bgId] → bpng.bin[0]。`bgId` 即 s.bin 的 `bA2`（j.java:1271 读取，1276 传入 `a.a(by4,by3,by2)`）。注意 **fpng 图号取自 p.bin 的 `fpngIdx`**（b.java:336，非 s.bin 直接给出），而 **bpng 恒用第 0 条**（b.java:350）。Boss(#6) 的 `bA2=bgId=1`（见第 6 节）正常映射到 b.bin 的真实条目 #1（原始 184B，去掉 8B 头后 A[] 176B = 8×22 背景表）——b.bin 有 2 条真实条目（count=3→真实=2，见第 0 节与第 3 节），bgId=1 非 EOF 哨兵。

## 11. 存疑清单（资源相关）

1. **a.bin `extId`（恒 17）、`reserved`（恒 [1,1,0,1]）、尾部 `trailCount/trail`** 语义未明（不影响解析，影响理解）。⚠️
