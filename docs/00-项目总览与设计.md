# 红魔特种兵 1 & 2 — 逆向复刻项目总览与设计

> 本文件既是**项目总览**，也是经确认的**复刻设计 spec**。所有解析文档、逆向中间产物、实现代码均沉淀在本仓库 `red-devil-re/` 下（red-devil 为项目名，仓库目录为 red-devil-re），便于 review 与后续二次开发。

## 1. 目标

把两款 2005/2006 年的 J2ME 功能机游戏 **尽可能原汁原味地**复刻到现代 Web 技术栈：**核心逻辑层做到位级一致**（随机数/资源解码/绘图时序逐行对应反编译源码），同时列明必要偏差（音频 MIDI 为设备相关近似、ASCII 字距/字体为设备相关近似、像素级逐帧自动 diff 因无周期精确模拟器而未做，详见第 6 节）。并在过程中沉淀完整的游戏解析文档，为后续二次开发迭代打基础。

| | 游戏1 红魔特种兵 | 游戏2 红魔特种兵2-深海战舰 |
|---|---|---|
| 包名 | `红魔特种兵.jar` (~95 KB) | `红魔特种兵2-深海战舰.jar` (~90 KB) |
| 发行 / 开发 | Sina / TGS | Sina |
| 目标机型 | Nokia N-Gage QD | Motorola E398 |
| 规范 | MIDP-1.0 / CLDC-1.0 | MIDP-1.0 / CLDC-1.0 |
| 引擎包 | `tjge`（混淆，13 个类） | `tjge`（混淆，12 个类） |
| 类型 | 横版射击/动作 | 动作 |
| 资源时间戳 | 2005-03 | 2006-02 |

两款游戏共用同一套自研引擎 `tjge`，资源打包格式一致（见 `bin资源格式.md`），因此先统一解析、提炼共性，再统一实现。

## 2. 关键决策（已确认）

1. **技术栈**：Web — TypeScript + HTML5 Canvas 2D。Canvas 的 `drawImage/fillRect/setClip/fillText` 与 J2ME `Graphics` 几乎 1:1 对应，逻辑可逐行迁移；跨平台、最适合二次开发迭代。
2. **精度**：核心逻辑层位级一致；音频(MIDI)/字体为设备相关近似，像素级逐帧自动 diff 待补（详见第 6 节）。
3. **顺序**：先全面解析两个游戏，再统一实现。
4. **产物沉淀**：反编译源码、重命名/注释后的可读源码、解码资源、解析文档等所有中间内容都保留在仓库内。

## 3. 复刻策略：J2ME 兼容层 + 逐行忠实移植

> 不凭"理解"重写，而是**逐行翻译**反编译出的逻辑。只要底层原语（随机数、绘图、资源读取、时序）与原版字节一致，上层逻辑就天然一致。

```
┌─────────────────────────────────────────────┐
│  Web 运行壳 (Vite)：键盘映射、画布缩放、主循环  │
├─────────────────────────────────────────────┤
│  忠实移植的游戏逻辑 (game1 / game2)            │  ← 逐行对应反编译源码
│  GameMIDlet + 各业务类（友好名 .ts，已语义化）  │
├─────────────────────────────────────────────┤
│  J2ME 兼容层 (j2me-shim)                      │  ← 复刻原语，保证位级一致
│  Graphics · Image · Random · Resource · Sound │
└─────────────────────────────────────────────┘
```

> 业务类编号：game1 为 `a`~`l`（12 个），game2 为 `a`~`k`（11 个）；两包另有 `index.ts` 入口，game2 还有 `jref.ts`（打破 ESM 循环依赖的基础设施叶子模块），均非业务类。类名与友好名映射见 `tools/rename-map.game{1,2}.json`。

### 位级一致的三个支点（均已验证可行）

1. **Random**：复刻 `java.util.Random` 的 48-bit 线性同余（`seed = (seed*0x5DEECE66D + 0xB) mod 2^48`）。本机 JDK 17 可生成参考序列做单测断言。原版用法见 `GameMIDlet`：构造时 `setSeed(System.currentTimeMillis())`，取随机数经 `GameMIDlet.a(int)`（`nextInt()` 取绝对值再取模）。
2. **资源解码**：`.bin` 归档格式已完全破解，提取的 PNG 与内嵌字节完全一致。
3. **绘图 / 时序**：Canvas 2D 与 J2ME `Graphics` 一一对应，主循环逐帧推进，复刻原版 `Thread.sleep` 帧节奏。
